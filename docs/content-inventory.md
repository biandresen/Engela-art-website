# Content Inventory

## Purpose

This document tracks the real content needed before the site can launch. A beautiful implementation will still feel unfinished if artwork data, images, contact details, or copy are missing.

Use this as a working checklist while collecting content from the artist.

## Site-Level Content

| Item | Status | Notes |
| --- | --- | --- |
| Artist name | Needed | Exact public name to show in header and page titles. |
| Site title | Needed | Usually artist name or artist name + art. |
| Short tagline | Needed | One short sentence for the home page. |
| Short intro | Needed | 2-4 sentences for home page. |
| Full artist bio | Needed | About page text. |
| Contact email | Needed | Email that receives buyer inquiries. |
| Public phone number | Optional | Only include if the artist wants phone contact. |
| Location/region | Optional | Useful for pickup/local context. |
| Social links | Optional | Instagram, Facebook, portfolio platforms, etc. |
| Artist portrait | Needed | Used on about page. |
| Studio/process images | Optional | Helps make the site more personal. |
| Privacy/contact text | Needed | Explain how inquiry data is used. |

## Language Decision

Choose one before writing final copy:

```txt
Norwegian-first
English-first
Bilingual
```

Default recommendation:

```txt
Norwegian-first for v1, with structure that can support English later.
```

Why:

- The first market is Norway.
- Norwegian copy supports trust with local buyers.
- Bilingual support can be added later if international demand appears.

## Image Checklist

For each painting, collect:

- Straight-on main image.
- Optional detail image showing texture.
- Optional room/context image.
- Image filename that is easy to understand.
- Alt text that describes the visual artwork.

Image quality requirements:

- Good lighting
- Accurate color
- Minimal glare
- Artwork edges visible when useful
- No distracting background unless intentionally styled

## Inquiry Email Content

Decide what the artist should receive in each inquiry email.

Recommended fields:

- Buyer name
- Buyer email
- Optional phone
- Message
- Painting title, if relevant
- Painting slug, if relevant
- Page URL, if available
- Submission timestamp

## Buyer Confirmation Content

Decide whether buyers should receive an automatic confirmation email.

V1 recommendation:

- Show an on-site success message first.
- Add buyer confirmation email later if needed.

Suggested success message:

```txt
Thank you for your inquiry. We have received your message and will get back to you as soon as possible.
```

## Privacy and Legal Copy

Because the site collects contact form data, include a short privacy note.

Minimum copy to prepare:

```txt
When you send an inquiry, we use your contact details only to respond to your message about the artwork. We do not sell your information.
```

This is not legal advice. Before launch, review whether a fuller privacy policy is needed based on the final hosting, analytics, and email tools.

## Launch Content Checklist

Before launch, confirm:

- Artist name is final.
- Home page tagline is final.
- Artist bio is final.
- Contact email is correct.
- At least a small set of paintings has complete data.
- Every visible painting has an image.
- Every image has alt text.
- Painting prices/statuses are correct.
- Inquiry success/error copy is written.
- Privacy/contact note is written.
- No placeholder text remains.

## Content Risks

- Missing prices or dimensions make buyer decisions harder.
- Poor image quality weakens trust.
- Unclear status can cause awkward sales conversations.
- Long unfocused copy can distract from the artwork.
- Missing privacy text is a problem when collecting personal data.
